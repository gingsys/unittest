Change Log

Proyecto: Sistema de Control de Acceso.
CONCAR

v. 0.0.16 - 05/03/2015

  • Se corrigió la forma como se graba los Workflows.
  • Se modificó la carga de Recursos, ahora se usa un Stored Procedure para hacerlo más rápido.
  • Se agregaron validacions del lado Cliente.
  • Se agregaron validacines del lado Servidor.
  • En Solicitud nueva ahora se muestra los recursos colapsados por defecto.

v. 0.0.14 - 26/02/2015

  • En Personas ahora se puede grabar SIN Area NI Cargo (No todas las personas trabajan en Concar ni tienen cargo).
  • En Personas ahora se muestra un mensaje adecuado si ya existe otra Persona con el mismo Tipo de Documento y Número de Documento..
  • En Personas ahora se muestra un mensaje adecuado si ya existe otra Persona con el mismo Número de ID interno.
  • En Persona Nueva ahora aparece por defecto Tipo de Documento = DNI.
  • Los Recursos ahora pueden tener un Area/Proyecto Asociado. Si se define un Area/Proyecto entonces el recurso solo aparecerá para las Personas que estén en dicha Area/Proyecto.
  • En Recursos ahora al seleccionar un Recurso Padre se hereda automáticamente la Categoría, el Tipo de Acceso y el Workflow.
  • En Solicitud se mejoró el layout de los subcontroles por recurso.
  • En Solicitud Nueva ahora aparece por defecto Prioridad = Normal.
  • En Solicitud Nueva se mejoró el tiempo que toma crear el árbol de Recursos.
  • En el editor de Workflow ahora se muestra una lista desplegable con el título "Solicitud", esta lista contiene los valores reemplazables de la solicitud.
  • Se modificó el ancho del Editor de Workflow.
  • Se agregó la columna de COMMON_VALUES_DEFAULT para definir una valor por defecto por CommonValueSet.
  • Se agregó un metodo GET a la Controladora de CommonValueSet para obtener el valor por defecto.

v. 0.0.5 - 05/02/2015

  • Se agregó el editor de texto para las Notificaciones de Workflow.
  • Se agregaron Areas y Cargos a valores Communes.
  • Se completo la grabación de Items de Workflow.
  • Se creó el servicio NotificationMailer para el envío de correos.
  • Se creó el servicio TaskManager para la ejecución de workflows y manejo de Timeouts.
  • Se agregó Jerarquía de Aprobación (BD, Controller y Cliente).
  • Se agregó System_Values (BD, Controller).
  • Se agregó Configuración de Notificaciones, que usa System_Values (Controller y Cliente).
  • Se agregaron las ventnas de búsqueda para Personas, Valores Comunes, Jerarquias de Aprobación, Workflow.
  • Se agregó la grabación de Cabecera de Solicituddes.

v. 0.0.4 - 23/01/2015

  • Se modificó el layout de la Aplicación para usar los estándares de Concar.
  • Se modificó la implementación de la definición de states(routes) para reflejar la estructura jerarquica del menú.
  • Se modificó la implementación del menú para hacer uso de rutas anidadas.
  • Modificación en el CSS para manejo de las ventanas de edición, cambiando de tablas a div.
  • Se crearon las rutas y ventana de Solicitud, la misma ventana deberá usarse para las 3 opciones: Mis Solicitudes Emitidas, Mis Solicitudes por Aprobar y Consulta de Solicitudes.
  • Se cambió el componente de grilla a ngGrid.

v. 0.0.3 - 21/01/2015

  • Implementado la Creación/Edición de Tipos de Acceso, sólo cabecera, falta el detalle (lista de valores).
  • Implementada la edición/creación de Categorías.
  • Implementada la edición/creación de Valores Comunes, sólo cabecera, falta el detalle (lista de valores).
  • Implementada la edición/creación de Workflows, sólo cabecera, falta el detalle (lista de items).
  • Implementada la ventana de Búsqueda de Tipos de Acceso
  • Implementada la ventana de Búsqueda de Categorías

v. 0.0.2 - 20/01/2015

  • Implementado en la Controladora (Servidor) las acciones de PUT y POST de AccessTypes y ResourceCategories.
  • Implementada la edición/creación de Categorías.
  • Implementado los eventos de la grilla de datos (external-scope).
  • Se implementó la clase AxsBaseRepository para centralizar ciertas operaciones de interfaz con las entidades.
